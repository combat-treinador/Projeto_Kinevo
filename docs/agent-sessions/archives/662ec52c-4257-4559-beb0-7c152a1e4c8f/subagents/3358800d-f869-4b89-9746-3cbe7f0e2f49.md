## Executive summary

There is **no explicit Firestore/API enum** like `workoutStyle`, `tipo_treino`, `modalidade`, or `estilo` on `BibliotecaTreinos` or on `exercicios[]`. Circuit vs strength is **inferred client-side** in Kinevo_Alunos from session title + synthesized tags. The only persisted signals are **text fields** (`metodo`, `dia_treino`, `rotulosDivisao`, `reps`, `intervalo`, `protocolo_cardio`, plan-level `tags`).

### Detection logic (`isCircuitSession`)

```10:24:f:/Projeto_Kinevo/Kinevo_Alunos/src/utils/circuitDetection.ts
const CIRCUIT_PATTERN =
  /CIRCUIT|CIRCUITO|HIIT|METABOL|FUNCIONAL|TABATA|AMRAP|EMOM/i;

export function isCircuitSession(session) {
  if (session.workoutStyle === "CIRCUIT") return true;
  const haystack = [session.titulo, ...(session.tags ?? [])]
    .filter(Boolean).join(" ").toUpperCase();
  return CIRCUIT_PATTERN.test(haystack);
}
```

**Important gap:** `exercicios[].metodo` (where Web stores `"TABATA"`, `"EMOM"`, etc.) is **not** used in `isCircuitSession()` or in `adaptBibliotecaToResponse()`.

### Hybrid blocks (`workoutBlocks.ts`)

Supports per-exercise `workoutStyle` and `tags`, but **`mapExercicio()` never populates them**, so block splitting usually inherits session-level `workoutStyle` unless mock/local data sets them.

### History (`historicoTreinos`)

`WorkoutHistoryPayload` / `saveCompletedWorkout()` do **not** persist `workoutStyle`. `periodization.service.ts` may read optional `modalidade` / `tipo` / `workoutStyle` from history if present, but the mobile app does not write them today.

## 4. Practical implication

To mark a division as circuit today, the trainer/Web layer must encode it in **text** the mobile app already scans:

1. Division label via `rotulosDivisao` or `dia_treino` containing e.g. `CIRCUITO`, `HIIT`, `FUNCIONAL`, `TABATA`, `EMOM`, `AMRAP`, or  
2. `foco` values that upper-case into matching `meta.tags` (e.g. foco = `"Funcional"`).

Prescribing `metodo: "TABATA"` on exercises **does not** currently flip `workoutStyle` to `"CIRCUIT"` in Kinevo_Alunos — that would require extending `adaptBibliotecaToResponse()` / `isCircuitSession()` to also inspect `metodo`, `reps`, or `intervalo`.
