### Ghost exercise risk matrix

| Layer | Can retain deleted/removed exercises? | Why |
|---|---|---|
| **`@kinevo_cached_plan` / `@kinevo_cached_today_plan`** | **HIGH** | Offline fallback serves stale full plans until server refresh; `_planSource.exercicios` preserved; only cleared on successful server fetch or schema bump |
| **`@kinevo_active_session` (Zustand)** | **HIGH** | Frozen `activeSessionCatalog.exercicios` survives app kill; reconciliation skipped when UI reads `fromCache: true` plans |
| **`listenerRegistry.lastResult`** | **MEDIUM** | In-memory stale snapshot until next Firestore emit; new mounts get old exercises instantly |
| **React Query `allActivePlans` / `todayPlan`** | **MEDIUM** | `staleTime: Infinity`; holds data until listener updates or invalidation; `gcTime` 5 min after unmount |
| **`@kinevo_offline_workouts`** | **LOW (historical)** | Stores past completed sessions, not live plan UI; survives logout |
| **Firestore `memoryLocalCache`** | **LOW** | Process-lifetime only |
| **`completedSessionIdsByPlan` (Zustand, not persisted)** | **None across restarts** | Session-only |

**Important gap:** `useAllActivePlans` only calls `reconcileWorkoutSessionWithPlans(rawPlans)` when `!plansFromCache`. If the app boots offline from AsyncStorage plan cache, **Zustand session catalog is not reconciled** until a realtime/server plan arrives.

**Logout clears:** React Query (all), Zustand session via `resetWorkoutProgress()` → `@kinevo_active_session`, auth scope cache, SecureStore tokens.  
**Logout does NOT clear:** plan AsyncStorage caches, offline workout queue.
