## 5. Root cause: why `intervalo` is dropped

### Primary cause — Firebase whitelist strips `intervalo`

Frontend sends `intervalo` inside each technique object, but **`normalizarTecnicas` never maps it**:

```2313:2320:F:/Projeto_Kinevo/Kinevo Pro/functions/index.js
        const normalizarTecnicas = (raw) => {
            if (!Array.isArray(raw)) return [];
            return raw.slice(0, 20).map((t) => ({
                metodo: coerceText(t?.metodo, 80),
                volume: coerceText(t?.volume, 40),
                serie: coerceText(t?.serie, 20),
                observacoes: coerceText(t?.observacoes, 220)
            }));
        };
```

After save + reload from Firestore, **`intervalo` is gone**. Exercise-level `intervalo` is preserved separately (**2332**).

### Secondary causes (frontend)

1. **Config. frequente** — `_configFreqTecnicaRow()` has no Série/Intervalo fields; `lerForm()` hardcodes `serie: ""` and omits `intervalo` (**4658–4663**). Applying a preset wipes those fields.
2. **Spreadsheet import** — `_tecnicasFromMetodoPlanilha()` (**1172–1184**) creates `{ metodo, volume, serie: "Todas", observacoes }` with **no `intervalo`**.
3. **Preview UI** — `BibliotecaView.js` (**236–244**) renders `metodo`, `volume`, `serie`, `observacoes` but **not `t.intervalo`**, so even in-memory values look missing in preview.

Recent UI work (`kinevo-biblioteca-modals.css?v=20260707_tec_intervalo` at **2341**) added Intervalo to the form, but **backend normalization was not updated**.

## Root cause: why technique `observacoes` may not be saved

Backend **does** persist technique `observacoes` (see `normalizarTecnicas` above). Loss happens on the **frontend paths**:

### A. Config. frequente round-trip

When saving/applying a frequent config, `lerForm()` reads technique rows from a **reduced UI** (metodo, volume, observacoes only). If the user re-saves a preset without re-entering technique observações, or applies an old preset saved before observações existed, **`observacoes` becomes `""`** via `_aplicarConfigFrequenteEmExercicio()` (**4284–4294**).

### B. MVC fallback zeroes all `tecnicas`

If `sincronizarEditorVisualParaJson()` is unavailable/empty, `BibliotecaController._readEditorRows()` hardcodes:

```49:49:F:/Projeto_Kinevo/Kinevo_v2/mvc/controllers/BibliotecaController.js
                tecnicas: []
```

That drops **all technique data**, including `observacoes`.

### C. Pre-OK re-render (mitigated)

Adding/removing techniques used to re-render before persisting DOM values. `_bindTecnicasFieldSync()` (**4449–4463**) and `_salvarEstadoModal()` before add/remove (**5705, 5717**) mitigate this for the main modal.

### D. Exercise vs technique observações

Exercise-level observações use `data-field="observacoes"` (textarea in `_renderExForm` **4942–4944**). Technique observações use `data-tec-field="observacoes"` — separate attributes, no collision.
