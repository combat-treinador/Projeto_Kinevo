## 1. Estrutura geral do projeto

O diretório `f:\Projeto_Kinevo\Kinevo Pro` é um projeto **Firebase full-stack** com empacotamento mobile via **Capacitor**. Não há README na raiz do projeto.

```

## 2. O que a aplicação faz

O **Kinevo PRO** (v2.0) é descrito como:

> *"Plataforma inteligente de Reabilitação e Performance"*

É uma ferramenta voltada a **profissionais de saúde e performance** (fisioterapeutas, médicos, treinadores) para:

- Coletar **anamnese clínica** e dados de treino
- Anexar **imagens, vídeos e PDFs** (exames, laudos)
- Gerar **laudos técnicos com IA** (postura, reabilitação, biomecânica, periodização)
- Montar e gerenciar **fichas de treino periodizadas** com editor visual
- Consultar um **mentor clínico em chat** com contexto da tela ativa

O fluxo típico: o profissional preenche formulários → envia mídias para Firebase Storage → a Cloud Function `perguntarAoOraculo` consulta base RAG (PDFs indexados) + Gemini → retorna laudo estruturado com tabelas, gráficos e recomendações.

Há também **visão computacional no cliente** (`KinevoPoseEngine`): MoveNet analisa vídeos/imagens para extrair landmarks, simetrógrafo, trilhos anatômicos e metadados cinemáticos enviados à IA.

## 3. Stack tecnológica

| Camada | Tecnologias |
|--------|-------------|
| **Frontend** | HTML5, CSS custom, JavaScript ES Modules, PWA (manifest + SW) |
| **Mobile** | Capacitor 8 (`@capacitor/android`, `app`, `keyboard`) — pacote `br.com.kinevopro.app` |
| **Backend** | Firebase Cloud Functions v2 (Node 22), `firebase-admin` |
| **Banco** | Cloud Firestore |
| **Arquivos** | Firebase Storage (PDFs, mídias) |
| **Auth** | Firebase Auth — login anônimo no Pro; login real no Kinevo_v2 |
| **IA** | Google Gemini 2.5 Flash (Vertex AI + API direta), embeddings `gemini-embedding-001` |
| **RAG** | PDFs → chunks vetorizados → coleção `Artigos` no Firestore |
| **Visão** | MoveNet / TensorFlow.js (client-side) |
| **Gráficos** | ApexCharts (CDN) |
| **PDF** | Export via `window.print()` (não html2pdf) |

**Firebase project:** `periodizacao-de-treinamento`  
**Região Functions:** `us-central1`  
**Hosting:** `periodizacao-de-treinamento.web.app`

**Autenticação no Kinevo Pro:** gate simples por senha em `sessionStorage` (`kinevoAutenticado = 'sim'`) + Firebase Auth anônimo para as Callables. Não é autenticação Firebase tradicional com e-mail/senha nesta variante.

## 4. Módulos e funcionalidades principais

### Abas do painel (`index.html`)

| Aba | Função |
|-----|--------|
| **Avaliações** | Avaliação postural e de padrões de movimento; comparativo antes/depois; simetrógrafo e trilhos anatômicos |
| **Reabilitação & Laudos** | Análise clínica, laudo completo ou resumido para paciente |
| **Performance & Treino** | Anamnese + periodização (mesociclos, valências, volume, equilíbrio muscular) |
| **Análise Biomecânica** | Laudo cinemático em 7 passos com metadados de vídeo |
| **Auditoria & Progressões** | Auditoria de progressões, valências alvo, fase pós-op |
| **Biblioteca de Treinos** | CRUD de fichas no Firestore, editor visual por semana/dia, dieta, compartilhamento |
| **Personalização (Workspace)** | Montagem customizável de ferramentas/campos (localStorage) |

### Chat flutuante: **Kinevo Sênior**
- Callable `conversarComOraculo`
- Lê contexto da tela ativa + RAG
- Mentor clínico para segunda opinião

### Cloud Functions (`functions/index.js`)

| Função | Papel |
|--------|-------|
| `lerArtigoNovo` | Trigger Storage: PDF → texto → embeddings → Firestore `Artigos` |
| `perguntarAoOraculo` | Núcleo IA: laudos por aba, RAG multi-pasta, visão, periodização |
| `gerenciarBibliotecaTreinos` | CRUD fichas (`BibliotecaTreinos`) |
| `gerenciarPerfisTreino` | Perfis de treino (`PerfisTreino`) |
| `conversarComOraculo` | Chat Sênior |
| `getTreinoCompartilhado` | Link público de ficha (`treino-share.html?token=...`) |

### Arquitetura MVC (`public/mvc/`)

Cada domínio tem **Model** (payload + upload), **View** (UI) e **Controller** (binding):
- `Avaliacoes`, `Reabilitacao`, `Performance`, `Biomecanica`, `Auditoria`, `Biblioteca`, `Personalizacao`

Utilitários importantes:
- `kinevoCallableGateway.js` — gateway único para Callables
- `kinevoPoseTrilhos.js` — pose + overlay de trilhos
- `kinevoMetadadosCinematicos.js` — análise de vídeo frame a frame
- `storageContext.js` — sincronização de contexto entre abas (ex.: Reabilitação → Performance)

### Coleções Firestore (regras em `firestore.rules`)

- `BibliotecaTreinos`, `PerfisTreino` — fichas e perfis (por `ownerUid`)
- `Artigos` — base RAG (leitura autenticada, escrita só via Functions)
- `perfisAlunos`, `tarefas`, `perfisUsuario`, `workouts` — referenciadas nas regras (usadas principalmente pelo Kinevo_v2)

### PWA + Android
- Instalável como app standalone
- Capacitor ajusta teclado e botão voltar no WebView Android
- Assets sincronizados em `android/app/src/main/assets/public/`

## 5. Conexão com outros projetos Kinevo

No workspace `f:\Projeto_Kinevo` coexistem pelo menos dois projetos irmãos:

### **Kinevo Pro** (este projeto)
- Variante **enxuta, mobile-first** (PWA + Capacitor Android)
- UI própria (sidebar escura, Nunito)
- **Backend completo** em `functions/`
- Auth: senha simples + Firebase anônimo
- Service Worker v4 (cache mínimo)

### **Kinevo_v2** (`f:\Projeto_Kinevo\Kinevo_v2`)
- Variante **web expandida** sobre template admin **Zono** (Bootstrap, FullCalendar, CKEditor)
- **Mesmo Firebase** (`periodizacao-de-treinamento`) — config idêntica em `features/firebaseConfig.js`
- **Não possui `functions/`** — consome o backend do Kinevo Pro
- Auth Firebase real (e-mail); redireciona anônimos para login
- Módulos extras não presentes no Pro:
  - `anamnese.html` / `anamnese-print.html`
  - `treino-share.html`
  - `exerciseProfileManager.js` (workouts por aluno)
  - `taskCalendarManager.js` (calendário/tarefas)
  - `headerProfileManager.js`, `whatsappMessageManager.js`
  - `clinicalCrossCheckManager.js` (Matriz de Confiança clínica)
  - `laudoAnteriorManager.js`, `pastaDropdownManager.js`
- Código MVC muito similar (models/controllers/views espelhados)
- Documentação interna em `docs/estabilizacao-kinevo.md` e regras Cursor em `.cursor/rules/`
- SW v135 com precache muito mais amplo

        Firebase: periodizacao-de-treinamento
        ├── Hosting (duas variantes de deploy)
        ├── Firestore (dados compartilhados)
        ├── Storage (PDFs RAG + mídias)
        └── Cloud Functions ← só em Kinevo Pro/functions/
```

**Evidências de compartilhamento:**
- Mesmo `projectId`, `apiKey`, `storageBucket`
- `functions/index.js` referencia campos do frontend Kinevo (`cruzamento_clinico_kinevo`, `KinevoPoseEngine`)
- `firestore.rules` menciona `exerciseProfileManager.js` (arquivo do Kinevo_v2)
- URL de compartilhamento aponta para `periodizacao-de-treinamento.web.app/treino-share.html` (página do v2)
- Índices Firestore (`workouts`, `perfisAlunos`, `tarefas`) alinhados ao v2

**Em resumo:** o **Kinevo Pro** é o núcleo backend + cliente mobile/PWA simplificado; o **Kinevo_v2** é a evolução web com CRM de alunos, calendário, anamnese dedicada e UX admin — ambos no mesmo ecossistema Firebase.
