### 2. How tabs/abas work

**CSS** (`index.html` lines **199вЂ“206**):
- All `.tab-content` inside `#conteudoPrincipal` are `display: none`
- Only `.tab-content.active` is `display: block`

**User click вЂ” `window.abrirAba`** (`app.js` lines **747вЂ“784**):
1. Removes `active` from all `.tab-content` and `.tab-button`
2. Adds `active` to the target panel (`document.getElementById(nomeDaAba)`)
3. Adds `active` to `event.currentTarget` (the clicked nav link)
4. Closes mobile sidebar if open
5. Special case: `abaPerformance` triggers `sincronizarPerformanceComLaudoReabilitacao()`

**Programmatic вЂ” `abrirAbaProgramaticamente(nomeIdAba)`** (`app.js` lines **348вЂ“379**):
- Same hide/show logic, but finds the matching `.tab-button` via `data-kinevo-tab` or `onclick` string match
- Used when navigating from Performance export, student profile edit, etc.

There is no `abaAtiva` global variable вЂ” the active tab is determined by `.tab-content.active` in the DOM.

### 4. Firestore collections for exercises

**There is no standalone exercise library collection.** No matches for `abaBibliotecaExercicios`, `exerciciosBanco`, or similar anywhere in the repo.

Exercises are stored as an **`exercicios` array field** inside documents in:

| Collection | Purpose | Access from frontend |
|---|---|---|
| **`BibliotecaTreinos`** | Training fichas (nome, pasta, objetivo, tags, **`exercicios[]`**, dieta, etc.) | `fnBiblioteca` в†’ Cloud Function `gerenciarBibliotecaTreinos` |
| **`PerfisTreino`** | Custom training profiles (linked to fichas) | `fnPerfis` |
| **`workouts`** | Student-assigned workouts (client-side Firestore in `exerciseProfileManager.js`) | Direct Firestore SDK |

Backend reference: `f:\Projeto_Kinevo\Kinevo Pro\functions\index.js` lines **2096+** вЂ” `db.collection("BibliotecaTreinos")`.

Frontend CRUD in `app.js` (section **3.2**, lines **1765вЂ“1903**):
- `chamarBibliotecaTreinos(action, body)` в†’ wraps `fnBiblioteca`
- Actions: `create`, `list`, `get`, `update`, `archive`, `duplicate`, `delete`, `create_share_link`
- Exercises are passed as `payload.exercicios` (JSON array), not as separate documents

MVC layer: `BibliotecaModel.js`, `BibliotecaController.js`, `BibliotecaView.js` (bootstrapped at `app.js` lines **6273вЂ“6284**).

### 6. How list tables are built

The app does **not** use jQuery DataTables for Kinevo feature lists. Theme assets include DataTables CSS/JS but they are unused in the main app logic.

**Biblioteca fichas list** вЂ” custom card HTML via `innerHTML`:
- `renderListaTreinosBiblioteca()` in `app.js` lines **1814вЂ“1852**
- Duplicate in `BibliotecaView.renderTreinos()` lines **27вЂ“57**
- Pattern: `.kinevo-treino-card` divs with `data-action` buttons (editar, duplicar, preview, etc.)

**Student workouts table** (`exerciseProfileManager.js` lines **265вЂ“310**):
- Static `<table>` in HTML with `#kex-tabela-tbody`
- Rows built via `document.createElement("tr")` + `innerHTML`

**AI result tables** (Performance/ReabilitaГ§ГЈo):
- Markdown rendered to HTML `<table>` elements
- Extracted via `extrairExerciciosDaTabelaPerformanceDom()` in `app.js`

**Preview/export tables**:
- Built as HTML strings (e.g. `app.js` line **5007**: `<table border="1"...>`)
